# public_release_core

Minimal public release for LEN resistance evidence synthesis.

## Scope
- Rewritten core workflow scripts only (no historical script versions).
- Minimal processed dataset bundle for reproducibility.
- Key raw source subset (`11 source records + key control files`).
- Source-to-claim mapping metadata for auditability.

## Directory Layout
- `code/`: rewritten core scripts.
- `data/processed/revision_v2/`: harmonized tables and core analysis outputs.
- `data/raw_key/`: key raw-source extraction files used by current claims.
- `data/manifests/`: provenance and manifest files.
- `metadata/`: source/bib and claim mapping tables.
- `results/`: outputs generated by rewritten scripts.

## Quick Start
1. Create environment and install dependencies.
2. Run scripts in order:

```bash
python code/01_build_harmonized_dataset.py
python code/02_model_comparison.py
python code/03_epistasis_and_context.py
python code/04_generate_main_figures.py
```

3. Check generated files in `results/`.

## Reproducibility Notes
- Current synthesis boundary: `26 total observations`, `23 complete-case` for model fitting.
- Cross-validation terminology: `leave-one-source-record-out (LOSO)`.
- Fitness panel uses `n=6` aggregated measurements.
- Mapping files:
  - `metadata/source_id_to_bibkey.csv`
  - `metadata/source_to_claim_mapping.csv`
  - `metadata/reproducibility_manifest.csv`
